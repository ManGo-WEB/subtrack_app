# SubTrack - Трекер подписок

Веб-приложение для учета и отслеживания регулярных подписок. Позволяет пользователям централизовать управление расходами, прогнозировать даты списаний и видеть общую сумму трат в единой валюте (RUB).

## Технологический стек

- **Frontend:** Next.js 15 (App Router), TypeScript, Tailwind CSS
- **UI Library:** Shadcn/UI (Radix Primitives)
- **Backend & Auth:** Supabase (PostgreSQL, GoTrue, Realtime)
- **Работа с датами:** date-fns
- **Иконки:** Lucide-React
- **Валидация:** React Hook Form + Zod
- **Уведомления:** Sonner

## Основные возможности

- ✅ Аутентификация через Supabase Auth (Email + Пароль или Magic Link)
- ✅ Переключатель видимости пароля в формах входа и регистрации
- ✅ Перевод всех сообщений об ошибках на русский язык
- ✅ Управление подписками (CRUD операции)
- ✅ Каталог популярных сервисов с автодополнением
- ✅ Создание приватных кастомных сервисов (видимы только создателю)
- ✅ Расчет дат следующего платежа
- ✅ Конвертация валют (USD, EUR → RUB) по курсу ЦБ РФ
- ✅ Расчет итоговой месячной суммы (lifetime подписки не учитываются)
- ✅ Отдельный итог для месячных подписок
- ✅ Сортировка подписок по дате следующего платежа
- ✅ Toast-уведомления с кнопкой закрытия (не исчезают автоматически)
- ✅ Адаптивный дизайн (Mobile-First) с минималистичным UI
- ✅ Отображение email пользователя в заголовке

## Установка и настройка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd subtrack_app
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка Supabase

1. Создайте проект в [Supabase](https://app.supabase.com)
2. Скопируйте `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. Создайте файл `.env.local` в корне проекта:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 4. Настройка базы данных

1. Откройте SQL Editor в Supabase Dashboard
2. Выполните миграции из папки `supabase/migrations/` **по порядку**:
   - `001_initial_schema.sql` - создание таблиц и RLS политик
   - `002_seed_services_catalog.sql` - заполнение каталога сервисов
   - `003_update_service_logos.sql` - обновление логотипов сервисов
   - `004_add_services_catalog_insert_policy.sql` - политика INSERT для каталога
   - `005_backfill_service_ids.sql` - заполнение service_id для существующих подписок
   - `006_cleanup_unused_services.sql` - очистка неиспользуемых сервисов (опционально)
   - `007_add_user_id_to_services_catalog.sql` - добавление user_id для приватных сервисов
   - `008_backfill_user_id_for_existing_services.sql` - привязка существующих сервисов к пользователям

### 5. Запуск приложения

```bash
npm run dev
```

Приложение будет доступно по адресу [http://localhost:3000](http://localhost:3000)

## Основные функции

### Управление подписками

- Добавление подписки с выбором сервиса из каталога или ручным вводом
- Создание кастомных сервисов (приватные, видны только создателю)
- Редактирование существующих подписок
- Удаление подписок с подтверждением
- Отображение даты следующего платежа
- Сортировка подписок по дате следующего платежа (ближайшие сверху)
- Визуальное выделение подписок с близкими датами платежа (< 3 дней) красной границей слева

### Расчеты

- Автоматический расчет даты следующего платежа на основе якорной даты и периода
- Конвертация валют в RUB по курсу ЦБ РФ (с пометкой "примерно" ~)
- Расчет итоговой месячной суммы (lifetime подписки не учитываются)
- Отдельный расчет суммы только для месячных подписок
- Кэширование курсов валют (обновление не чаще 1 раза в сутки)

### Безопасность

- Row Level Security (RLS) политики в Supabase
- Пользователи видят только свои подписки
- Приватные кастомные сервисы видны только создателю
- Предустановленные сервисы (user_id = NULL) видны всем пользователям
- Автоматическое создание профиля при регистрации

### Уведомления и UX

- Toast-уведомления для успешных операций и ошибок
- Все ошибки переводятся на русский язык автоматически
- Уведомления не исчезают автоматически, пользователь закрывает их вручную
- Переключатель видимости пароля в формах авторизации и регистрации
- Улучшенные визуальные блоки ошибок в формах
